# Markdown Notes RAG with LangChain and Hugging Face

This is a small learning project for RAG: retrieval augmented generation.

You will:

1. Put markdown notes in `notes/`.
2. Embed those notes with the Hugging Face API.
3. Store the vectors locally in Chroma.
4. Retrieve relevant note chunks for a question.
5. Ask a Hugging Face-hosted language model to answer using those chunks.

## 1. Create a virtual environment

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## 2. Add your Hugging Face token

Copy the example environment file:

```bash
cp .env.example .env
```

Then edit `.env` and set your token:

```bash
HF_TOKEN=hf_your_token_here
HUGGINGFACEHUB_API_TOKEN=hf_your_token_here
```

For this project, the token only needs the fine-grained permission:

```text
Make calls to Inference Providers
```

## 3. Add your notes

Put your own `.md` files in `notes/`.

The included `notes/example-rag.md` file is just there so you can test the
pipeline before adding your real notes.

## 4. Build the vector index

```bash
python rag_notes.py index --reset
```

This reads `notes/*.md`, splits the notes into chunks, embeds each chunk through
Hugging Face, and stores the vectors in:

```text
work/chroma_notes/
```

## 5. Inspect retrieval before using the LLM

This is the best way to learn what RAG is doing:

```bash
python rag_notes.py search "Why does chunking matter?"
```

If the retrieved chunks are bad, the final answer will probably be bad too.
RAG quality starts with retrieval.

## 6. Ask a question

```bash
python rag_notes.py ask "What are the main steps in RAG?"
```

The answer is generated by the HF language model, but it receives your retrieved
note chunks as context.

## How The Code Maps To RAG

| RAG step | Function |
| --- | --- |
| Load markdown | `load_markdown_notes()` |
| Split into chunks | `split_documents()` |
| Embed chunks | `make_embeddings()` |
| Store vectors | `build_index()` |
| Retrieve chunks | `search_notes()` / `answer_question()` |
| Generate answer | `make_chat_model()` / `answer_question()` |

## Good Things To Experiment With

- Change `chunk_size` and `chunk_overlap` in `split_documents()`.
- Change `--k` when searching or asking.
- Try a different `HF_EMBED_MODEL`.
- Try a different `HF_LLM_MODEL`.
- Ask questions where the answer is not in your notes and see whether the model
  admits it does not know.

## Common Issues

If the embedding call fails, check:

- Your token is set in `.env`.
- The token has `Make calls to Inference Providers`.
- The embedding model supports `feature-extraction`.
- Your free Hugging Face credits have not run out.

If the final answer call fails, try another text-generation model in `.env`.
Different Hugging Face providers expose different models and tasks.
